# Financial modeling guide (competition-grade)

## 1. Clapp-style gross profit

For each year \(N\) (or each period you present):

**Year N sales × (price per sale − cost per sale) = Year N gross profit**

Equivalently: **Units × (Price − COGS) = Gross Profit**

State **units**, **price**, **COGS**, and **assumptions** explicitly (conversion rate, attach rate, seasonality if material).

## 2. Assumptions tables (minimum credible set)

Build a small table founders can defend in Q&A:

| Assumption | Value | Source / how you’ll validate | Sensitivity (what breaks the plan) |
|------------|-------|------------------------------|-----------------------------------|
| Price | | | |
| COGS / unit | | | |
| Units or conversion | | | |
| CAC (if paid acquisition) | | | |
| Churn / repeat purchase | | | |

Rule: if you can’t cite **how you know**, label it a **hypothesis** and name the next experiment.

## 3. Common traps

- **Ignoring CAC** while projecting aggressive growth.  
- **Underestimating COGS** (packaging, returns, payment fees, support burden).  
- **Hockey-stick revenue** without channel or pipeline logic.  
- **Confusing margin with cash** (working capital, inventory, payment terms).  
- **Top-down only** (“1% of TAM”) with no bottom-up build.

## 4. Challenger prompts (red-team for numbers)

Use in Q&A prep when a figure sounds too clean or a judge will dig for provenance:

- **“Show the interview log or data trail behind that conversion rate.”**  
- **“What was the sample size, who was excluded, and what’s the selection bias?”**  
- **“If we cut this revenue line by 50%, what breaks first—unit economics or the story?”**

## 5. Break-even basics

- **Contribution margin per unit** = Price − variable cost per unit.  
- **Break-even units** = fixed costs / contribution margin per unit (hold assumptions explicit; flag if oversimplified).  
- Use break-even to force conversation about **what must be true** operationally.

## 6. Use of funds frameworks

- **Milestone-based**: each tranche tied to a proof (prototype, pilot, channel test).  
- **Category caps**: product, marketing, inventory, legal/IP, ops — with **%** and **$**.  
- **Runway thinking**: months of runway implied by burn (even rough) beats vague “growth.”

## 7. Revenue model archetypes

- **Subscription**: MRR/ARR, churn, expansion; trials and payment friction.  
- **Transaction**: take rate, GMV vs revenue, dispute/refund costs.  
- **Marketplace**: supply + demand acquisition, liquidity, leakage.  
- **Freemium**: conversion hypothesis; support cost of free users.  
- **Licensing**: deal size, cycle length, implementation burden.

## 8. Conservative vs base vs optimistic

Present **three scenarios** as ranges, not as three unrelated dreams:

- **Conservative**: fewest assumptions; easiest to defend in competition.  
- **Base**: requires one or two **proven** conversion or channel facts.  
- **Optimistic**: only if you can name **what must go right** (partnership, viral loop, regulatory clearance).

Coach founders to **lead with conservative** and show what evidence upgrades the scenario.
